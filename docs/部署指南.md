## 快速开始

### 方式一：使用 docker-compose（推荐）

```bash
# 1. 克隆仓库
git clone <repository-url>
cd 答题网站

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 修改 SECRET_KEY、ADMIN_PASSWORD、POSTGRES_PASSWORD 等敏感配置

# 3. 启动服务（自动初始化数据库和管理员）
docker-compose up -d

# 4. 访问应用
# http://localhost:5002
```

### 方式二：本地开发

```bash
# 1. 克隆仓库
git clone <repository-url>
cd 答题网站

# 2. 配置环境变量
cp .env.example .env
# 修改 .env: FLASK_ENV=development, FLASK_DEBUG=true, DATABASE_URL=sqlite:///instance/exam.db

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动服务（自动初始化数据库和管理员）
python run.py
```

### 默认账号

系统首次启动时会自动创建管理员账户：

- **用户名**: `admin`（可通过 `ADMIN_USERNAME` 环境变量修改）
- **密码**: `admin`（可通过 `ADMIN_PASSWORD` 环境变量修改）
- **邮箱**: `admin@example.com`（可通过 `ADMIN_EMAIL` 环境变量修改）
- ⚠️ **生产环境请务必通过环境变量修改默认密码！**

## Docker Hub 部署

### 方式一：命令行传递环境变量（推荐）

```bash
docker run -d \
  -e SECRET_KEY="$(openssl rand -base64 32)" \
  -e ADMIN_USERNAME="admin" \
  -e ADMIN_PASSWORD="your-secure-password" \
  -e ADMIN_EMAIL="admin@example.com" \
  -e DATABASE_URL="postgresql://user:password@host:5432/dbname" \
  -p 5002:5002 \
  your-username/exam-system:latest
```

### 方式二：使用环境变量文件

```bash
# 创建环境变量文件
cat > exam-env.conf << 'EOF'
SECRET_KEY=your-secret-key
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password
ADMIN_EMAIL=admin@example.com
DATABASE_URL=postgresql://user:password@host:5432/dbname
EOF

# 使用环境变量文件运行
docker run -d \
  --env-file exam-env.conf \
  -p 5002:5002 \
  your-username/exam-system:latest
```

### 方式三：使用 docker-compose

```bash
# 1. 创建 docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  exam-system:
    image: your-username/exam-system:latest
    ports:
      - "5002:5002"
    environment:
      SECRET_KEY: your-secret-key
      ADMIN_USERNAME: admin
      ADMIN_PASSWORD: your-password
      ADMIN_EMAIL: admin@example.com
      DATABASE_URL: postgresql://user:password@db:5432/examdb
    depends_on:
      - db

  db:
    image: postgres:14-alpine
    environment:
      POSTGRES_DB: examdb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
EOF

# 2. 启动服务
docker-compose up -d
```

### 从 Docker Hub 下载配置模板

```bash
# 下载配置模板
wget https://raw.githubusercontent.com/your-repo/main/.env.example

# 修改配置
vim .env

# 运行容器
docker run -d --env-file .env -p 5002:5002 your-username/exam-system:latest
```

## 环境准备

### 1. 安装 Docker

#### Ubuntu/Debian

```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo apt-get install docker-compose-plugin

# 添加用户到 docker 组
sudo usermod -aG docker $USER
newgrp docker
```

#### CentOS/RHEL

```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### macOS

```bash
# 使用 Homebrew 安装
brew install --cask docker

# 启动 Docker Desktop
open /Applications/Docker.app
```

#### Windows

下载并安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)

### 2. 验证安装

```bash
docker --version
docker-compose --version
docker run hello-world
```

## Docker部署

### 基本部署

```bash
# 1. 克隆项目
git clone <repository-url>
cd 答题网站

# 2. 配置环境变量
cp .env.example .env
```

如需修改端口，编辑 `docker-compose.yml`：


## 生产环境部署

### 生产环境配置

1. **修改 .env 文件**

```bash
# 生产环境配置示例
FLASK_ENV=production
FLASK_DEBUG=false
SECRET_KEY=your-strong-secret-key-here
ADMIN_PASSWORD=your-admin-password
POSTGRES_PASSWORD=your-db-password
```

2. **使用 PostgreSQL 数据库**

```bash
# 确保 docker-compose.yml 中配置了 PostgreSQL 服务
DATABASE_URL=postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}
```

3. **配置反向代理（Nginx）**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:5002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

4. **配置 HTTPS（Let's Encrypt）**

```bash
# 使用 certbot 获取 SSL 证书
sudo certbot --nginx -d your-domain.com
```

### 性能优化

1. **调整 Gunicorn 配置**

```yaml
# 在 docker-compose.yml 中
environment:
  GUNICORN_WORKERS: 4        # 根据CPU核心数调整
  GUNICORN_THREADS: 2       # 每个worker的线程数
```

2. **启用缓存**

```bash
# 添加 Redis 服务
docker-compose.yml 中添加 redis 服务
```

## 环境变量配置

### 必需配置

| 变量名 | 说明 | 默认值 | 是否必需 |
|--------|------|--------|---------|
| `SECRET_KEY` | Flask 安全密钥 | dev-secret-key | 是（生产环境） |
| `ADMIN_PASSWORD` | 管理员密码 | admin | 是（生产环境） |
| `DATABASE_URL` | 数据库连接 | sqlite:///instance/exam.db | 是 |
| `POSTGRES_PASSWORD` | PostgreSQL 密码 | exam_password_123456 | Docker部署 |

### 可选配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `ADMIN_USERNAME` | 管理员用户名 | admin |
| `ADMIN_EMAIL` | 管理员邮箱 | admin@example.com |
| `FLASK_ENV` | Flask 环境 | production |
| `FLASK_DEBUG` | 调试模式 | false |
| `LOG_LEVEL` | 日志级别 | INFO |
| `MAX_CONTENT_LENGTH` | 最大上传大小 | 52428800 (50MB) |

### 配置优先级

```
命令行环境变量 > .env 文件 > docker-compose.yml 默认值 > 代码默认值
```

## 数据库管理

### 自动初始化

系统首次启动时会自动完成：
1. 创建数据库表结构
2. 执行数据库迁移
3. 创建管理员账户

### 数据库备份

```bash
# 备份 PostgreSQL 数据库
docker-compose exec db pg_dump -U examuser examdb > backup.sql

# 备份 SQLite 数据库
cp instance/exam.db backup/exam_$(date +%Y%m%d).db
```

### 数据库恢复

```bash
# 恢复 PostgreSQL 数据库
docker-compose exec -T db psql -U examuser examdb < backup.sql

# 恢复 SQLite 数据库
cp backup/exam_20240311.db instance/exam.db
```

## 故障排查

### 常见问题

#### 1. 容器无法启动

```bash
# 查看日志
docker-compose logs web

# 检查端口占用
lsof -i :5002
```

#### 2. 数据库连接失败

```bash
# 检查数据库状态
docker-compose ps db

# 查看数据库日志
docker-compose logs db

# 测试数据库连接
docker-compose exec db psql -U examuser -d examdb
```

#### 3. 权限问题

```bash
# 修复上传目录权限
sudo chown -R $USER:$USER uploads logs instance
```

#### 4. 管理员无法登录

```bash
# 检查管理员账户
docker-compose exec web python -c "
from app import create_app
from app.models.user import User
app = create_app()
with app.app_context():
    admin = User.get_by_username('admin')
    if admin:
        print(f'管理员存在: {admin.username}, 邮箱: {admin.email}')
    else:
        print('管理员不存在')
"

# 重置管理员密码（删除后重启自动创建）
docker-compose exec db psql -U examuser -d examdb -c "DELETE FROM users WHERE username='admin';"
docker-compose restart web
```

### 日志查看

```bash
# 实时查看所有日志
docker-compose logs -f

# 查看最近 100 行日志
docker-compose logs --tail=100 web

# 查看应用日志文件
tail -f logs/exam_system.log
```

## 备份与恢复

### 自动备份脚本

创建备份脚本 `backup.sh`：

```bash
#!/bin/bash

BACKUP_DIR="/path/to/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
docker-compose exec -T db pg_dump -U examuser examdb > $BACKUP_DIR/examdb_$DATE.sql

# 备份上传文件
tar -czf $BACKUP_DIR/uploads_$DATE.tar.gz uploads/

# 删除 30 天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: examdb_$DATE.sql, uploads_$DATE.tar.gz"
```

### 定时备份（Cron）

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点执行备份
0 2 * * * /path/to/backup.sh >> /var/log/exam-backup.log 2>&1
```
