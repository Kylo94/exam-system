## 快速开始

### 一键部署

```bash
# 1. 克隆仓库
git clone <repository-url>
cd 答题网站

# 2. 配置环境变量
cp .env.example .env

# 3. 启动服务
docker-compose up -d

# 4. 访问应用
# http://localhost:5002
```

## 环境准备

### 1. 安装 Docker

#### Ubuntu/Debian

```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo apt-get install docker-compose-plugin

# 添加用户到 docker 组
sudo usermod -aG docker $USER
newgrp docker
```

#### CentOS/RHEL

```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### macOS

```bash
# 安装 Docker Desktop
# https://www.docker.com/products/docker-desktop

# 或使用 Homebrew
brew install --cask docker
```

#### Windows

```bash
# 安装 Docker Desktop
# https://www.docker.com/products/docker-desktop
# 确保 WSL2 已启用
```

### 2. 验证安装

```bash
docker --version
docker-compose --version
docker ps
```

## Docker部署

### 方式一：使用 Docker Compose（推荐）

```bash
# 1. 配置环境变量
cp .env.example .env

# 2. 编辑配置（可选）
nano .env

# 3. 构建并启动
docker-compose up -d

# 4. 查看日志
docker-compose logs -f web

# 5. 验证部署
docker-compose ps
```

### 方式二：自定义配置

```bash
# 设置环境变量
export SECRET_KEY="$(python -c "import secrets; print(secrets.token_hex(32))")"
export ADMIN_PASSWORD="your-password"
export POSTGRES_PASSWORD="your-db-password"

# 启动服务
docker-compose up -d
```

### 环境变量配置

| 变量名 | 默认值 | 说明 | 是否必须修改 |
|-------|--------|------|------------|
| SECRET_KEY | dev-secret-key | Flask密钥 | ✅ 生产环境 |
| ADMIN_USERNAME | admin | 管理员用户名 | 可选 |
| ADMIN_PASSWORD | admin | 管理员密码 | ✅ 生产环境 |
| ADMIN_EMAIL | admin@example.com | 管理员邮箱 | 可选 |
| POSTGRES_DB | examdb | 数据库名称 | 可选 |
| POSTGRES_USER | examuser | 数据库用户 | 可选 |
| POSTGRES_PASSWORD | exam_password_123456 | 数据库密码 | ✅ 生产环境 |

### 验证部署

```bash
# 检查服务状态
docker-compose ps

# 检查健康状态
docker inspect exam-web | grep -A 10 Health

# 检查初始化日志
docker-compose logs web | grep "✓"
```

## 本地开发部署

### 方式一：直接运行

```bash
# 1. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate  # Windows

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env

# 4. 启动应用（自动初始化数据库和管理员）
python run.py

# 5. 访问应用
# http://localhost:5002
```

### 方式二：使用 Flask CLI

```bash
# 1. 设置环境变量
export FLASK_APP=run.py
export FLASK_ENV=development

# 2. 初始化数据库
flask db upgrade

# 3. 启动开发服务器
flask run --host=0.0.0.0 --port=5002
```

### 开发环境特性

✅ 热重载：代码修改自动重启
✅ 调试模式：显示详细错误信息
✅ SQLite 数据库：无需额外数据库服务
✅ 简单配置：最小化配置即可运行

## 生产环境部署

### 快速部署

```bash
# 1. 克隆项目
git clone <repository-url>
cd 答题网站

# 2. 配置环境变量
cp .env.example .env
nano .env

# 3. 生成安全密钥
python -c "import secrets; print(secrets.token_hex(32))"

# 4. 构建镜像
docker-compose build

# 5. 启动服务
docker-compose up -d

# 6. 验证部署
docker-compose logs web | grep "✓"
```

### 高级配置

#### 1. Gunicorn 配置

编辑 `Dockerfile` 中的启动命令：

```dockerfile
CMD ["gunicorn", "--bind", "0.0.0.0:5002",
      "--workers", "4",           # Worker 进程数
      "--threads", "2",            # 每个 Worker 的线程数
      "--timeout", "120",          # 超时时间
      "--access-logfile", "-",      # 访问日志输出到标准输出
      "--error-logfile", "-",       # 错误日志输出到标准输出
      "--log-level", "info",       # 日志级别
      "wsgi:app"]
```

#### 2. PostgreSQL 配置

编辑 `docker-compose.yml`：

```yaml
db:
  image: postgres:14-alpine
  command: >
    postgres
    -c max_connections=200
    -c shared_buffers=256MB
    -c effective_cache_size=1GB
    -c maintenance_work_mem=64MB
    -c checkpoint_completion_target=0.9
    -c wal_buffers=16MB
```

#### 3. 数据持久化

```yaml
volumes:
  postgres_data:  # PostgreSQL 数据
  uploads:        # 上传文件
  logs:          # 日志文件
```

### 安全配置

#### 1. 修改默认密码

```bash
# 生成强密码
openssl rand -base64 32

# 编辑 .env 文件
POSTGRES_PASSWORD=<生成的密码>
ADMIN_PASSWORD=<生成的密码>
SECRET_KEY=<生成的密钥>
```

#### 2. 配置 HTTPS（可选）

```bash
# 使用 Let's Encrypt
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

#### 3. 配置防火墙

```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

## 数据库管理

### 数据库迁移

```bash
# 查看当前版本
flask db current

# 查看迁移历史
flask db history

# 创建新迁移
flask db migrate -m "描述变更"

# 应用迁移
flask db upgrade

# 回滚迁移
flask db downgrade
```

### 数据库备份

```bash
# 备份数据库
docker-compose exec db pg_dump -U examuser examdb > backup_$(date +%Y%m%d_%H%M%S).sql

# 压缩备份
docker-compose exec db pg_dump -U examuser examdb | gzip > backup_$(date +%Y%m%d_%H%M%S).sql.gz
```

### 数据库恢复

```bash
# 从 SQL 文件恢复
docker-compose exec -T db psql -U examuser examdb < backup_20260311_120000.sql

# 从压缩文件恢复
gunzip < backup_20260311_120000.sql.gz | docker-compose exec -T db psql -U examuser examdb
```

### 连接数据库

```bash
# 进入数据库容器
docker-compose exec db psql -U examuser -d examdb

# 查看数据库列表
\l

# 查看表列表
\dt

# 退出
\q
```

### 数据库性能优化

```sql
-- 查看表大小
SELECT
  schemaname,
  tablename,
  pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 查看慢查询
SELECT * FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;

-- 重建索引
REINDEX DATABASE examdb;

-- 清理数据库
VACUUM FULL;
ANALYZE;
```

## 故障排查

### 常见问题

#### 1. 容器无法启动

```bash
# 查看详细日志
docker-compose logs web

# 检查端口占用
lsof -i :5002

# 清理并重启
docker-compose down -v
docker-compose up -d
```

#### 2. 数据库连接失败

```bash
# 检查数据库状态
docker-compose ps db

# 等待数据库就绪
docker-compose exec db pg_isready

# 检查环境变量
docker-compose exec web env | grep DATABASE_URL
```

#### 3. 权限错误

```bash
# 修复目录权限
chmod -R 755 uploads logs instance

# 修改所有者
chown -R $USER:$USER uploads logs instance
```

#### 4. 内存不足

```bash
# 减少 Worker 数量
# 编辑 Dockerfile
CMD ["gunicorn", "--bind", "0.0.0.0:5002", "--workers", "2", ...]

# 增加系统交换空间
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

#### 5. Nginx 502 Bad Gateway

```bash
# 检查 Web 服务
docker-compose ps web

# 检查 Nginx 配置
docker-compose exec nginx nginx -t

# 重启 Nginx
docker-compose restart nginx
```

### 日志查看

```bash
# 查看所有服务日志
docker-compose logs

# 查看 Web 应用日志
docker-compose logs -f web

# 查看数据库日志
docker-compose logs -f db

# 查看最近 100 行
docker-compose logs --tail=100 web
```

### 健康检查

```bash
# 检查容器健康状态
docker-compose ps

# 手动测试
curl http://localhost:5002/

# 检查数据库健康
docker-compose exec db pg_isready -U examuser -d examdb
```

## 备份与恢复

### 自动备份

创建备份脚本 `scripts/backup.sh`：

```bash
#!/bin/bash

BACKUP_DIR="./backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份数据库
docker-compose exec db pg_dump -U examuser examdb | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 备份上传文件
tar -czf $BACKUP_DIR/uploads_$DATE.tar.gz uploads/

# 备份配置文件
cp .env $BACKUP_DIR/.env_$DATE

echo "备份完成: $BACKUP_DIR"

# 删除7天前的备份
find $BACKUP_DIR -type f -mtime +7 -delete
```

设置定时任务：

```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨 2 点备份
0 2 * * * /path/to/答题网站/scripts/backup.sh >> /var/log/backup.log 2>&1
```

### 手动备份

```bash
# 备份数据库
docker-compose exec db pg_dump -U examuser examdb > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份上传文件
tar -czf uploads_backup_$(date +%Y%m%d_%H%M%S).tar.gz uploads/
```

### 数据恢复

```bash
# 从 SQL 文件恢复
docker-compose exec -T db psql -U examuser examdb < backup_20260311_120000.sql

# 从压缩文件恢复
gunzip < backup_20260311_120000.sql.gz | docker-compose exec -T db psql -U examuser examdb

# 恢复上传文件
tar -xzf uploads_backup_20260311_120000.tar.gz
```
