# 日志系统说明

## 概述

在线答题系统已配置完整的日志系统，支持开发和生产环境的日志记录。

## 日志配置

### 开发环境
- **日志级别**: DEBUG
- **输出位置**: 
  - 控制台（实时输出）
  - 文件（`logs/exam_system.log`）
- **文件大小**: 单个文件最大 10MB
- **备份数量**: 最多保留 5 个备份文件

### 生产环境
- **日志级别**: INFO
- **输出位置**: 文件（`logs/exam_system.log`）
- **文件大小**: 单个文件最大 10MB
- **备份数量**: 最多保留 10 个备份文件

## 日志格式

```
%(asctime)s %(levelname)s: %(message)s [in %(pathname)s:%(lineno)d]
```

示例：
```
2026-02-10 08:55:23,456 INFO: 在线答题系统启动（开发模式） [in /path/to/app/__init__.py:15]
2026-02-10 08:55:24,789 DEBUG: 执行数据库查询 [in /path/to/app/services/question_service.py:123]
2026-02-10 08:55:25,123 ERROR: 用户登录失败 [in /path/to/app/routes/auth.py:45]
```

## 日志目录结构

```

├── exam_system.log.2           # 第2个备份

```

## 在代码中使用日志

### 基本用法

```python
from flask import current_app

# 获取logger
logger = current_app.logger

# 不同级别的日志
logger.debug('调试信息')
logger.info('普通信息')
logger.warning('警告信息')
logger.error('错误信息')
logger.critical('严重错误')
```

### 在服务类中使用

```python
from app.services.base import BaseService

class QuestionService(BaseService):
    def create_question(self, **kwargs):
        try:
            logger = self.get_logger()  # BaseService提供的logger
            
            logger.debug(f'创建题目: {kwargs}')
            
            question = self.create(kwargs)
            
            logger.info(f'题目创建成功，ID: {question.id}')
            return question
            
        except Exception as e:
            logger.error(f'创建题目失败: {str(e)}', exc_info=True)
            raise
```

### 在路由中使用

```python
from flask import Blueprint, jsonify

questions_bp = Blueprint('questions', __name__)

@questions_bp.route('/questions', methods=['POST'])
def create_question():
    try:
        logger = questions_bp.app.logger
        
        data = request.get_json()
        logger.info(f'收到创建题目请求: {data}')
        
        # 处理逻辑...
        
        return jsonify({'success': True})
        
    except Exception as e:
        logger.error(f'创建题目错误: {str(e)}', exc_info=True)
        return jsonify({'success': False, 'message': str(e)}), 500
```

## 日志级别说明

| 级别 | 说明 | 使用场景 |
|------|------|----------|
| DEBUG | 最详细的调试信息 | 变量值、函数调用、数据库查询等 |
| INFO | 一般信息 | 用户操作、系统状态变化、重要事件 |
| WARNING | 警告信息 | 潜在问题、已处理的异常、性能警告 |
| ERROR | 错误信息 | 捕获的异常、操作失败 |
| CRITICAL | 严重错误 | 系统崩溃、致命错误、需要立即处理的问题 |

## 日志轮转

日志系统使用 `RotatingFileHandler` 进行自动轮转：

- 当日志文件达到 10MB 时自动轮转
- 轮转后的文件重命名为 `exam_system.log.1`, `exam_system.log.2` 等
- 超过备份数量的旧文件会被自动删除

## 最佳实践

### 1. 合理选择日志级别

```python
# ✅ 推荐
logger.debug(f'查询参数: {params}')  # 开发时使用，生产环境可能不会记录
logger.info(f'用户 {user_id} 创建了考试 {exam_id}')  # 重要业务操作
logger.warning(f'API调用超过阈值: {response_time}ms')  # 潜在性能问题
logger.error(f'数据库连接失败: {str(e)}', exc_info=True)  # 错误信息，包含堆栈

# ❌ 不推荐
logger.info(f'每个HTTP请求')  # 太频繁，会产生大量日志
logger.debug(f'整个响应内容: {large_data}')  # 不要记录敏感或大量数据
```

### 2. 保护敏感信息

```python
# ✅ 推荐
logger.info(f'用户登录: user_id={user_id}')
logger.debug(f'密码哈希: {password_hash}')

# ❌ 不推荐
logger.info(f'用户登录: username={username}, password={password}')  # 不要记录明文密码
logger.info(f'API密钥: {api_key}')  # 不要记录敏感密钥
```

### 3. 使用结构化日志

对于复杂的日志，建议使用结构化格式：

```python
import json

logger.info(json.dumps({
    'event': 'user_action',
    'user_id': user_id,
    'action': 'create_exam',
    'exam_id': exam_id,
    'timestamp': datetime.now().isoformat()
}))
```

### 4. 异常处理时记录完整堆栈

```python
try:
    # 可能出错的代码
    pass
except Exception as e:
    # 记录完整堆栈信息
    logger.error(f'操作失败: {str(e)}', exc_info=True)
    # 或者
    logger.exception('操作失败')
```

## 故障排查

### 问题1：日志文件没有生成

**原因**：
- logs目录没有创建权限
- 应用没有正确调用 `init_app`

**解决**：
```bash
# 手动创建logs目录
mkdir -p logs
chmod 755 logs

# 检查app初始化是否正确
# app/__init__.py 中应该有:
# config.init_app(app)
```

### 问题2：日志文件过大

**原因**：
- 日志级别设置过低（如DEBUG）
- 日志记录过于频繁

**解决**：
```python
# 生产环境使用INFO或WARNING级别
file_handler.setLevel(logging.WARNING)

# 减少不必要的日志
# logger.debug(...)  # 避免频繁使用DEBUG级别
```

### 问题3：日志没有包含预期内容

**原因**：
- 日志级别过高，过滤掉了信息
- 日志handler未正确配置

**解决**：
```python
# 降低日志级别
app.logger.setLevel(logging.DEBUG)

# 确认handler配置
print(app.logger.handlers)  # 应该包含RotatingFileHandler
```

## 注意事项

1. **日志文件不提交到Git**：`.gitignore` 已配置忽略 `.log` 文件
2. **敏感信息保护**：不要记录密码、密钥、个人隐私等信息
3. **性能考虑**：避免在高并发场景下记录大量DEBUG日志
4. **磁盘空间**：定期清理旧的日志备份文件
5. **时区问题**：日志时间戳使用系统时区，建议配置UTC
