# 部署检查清单

本文档用于验证从 git clone 开始到完成部署的整个流程。

## 1. Git Clone 检查

### 检查点
- [ ] 仓库可以正常克隆
- [ ] 克隆后所有必要的文件都存在
- [ ] .gitignore 配置正确

### 关键文件
```
.gitignore              # 正确排除 .env、instance/、uploads/ 等敏感/临时文件
Dockerfile              # Docker 镜像构建配置
docker-compose.yml      # Docker Compose 部署配置
requirements.txt        # Python 依赖
run.py                  # 开发环境启动入口
wsgi.py                 # 生产环境启动入口
app/__init__.py         # 应用工厂 + 自动初始化逻辑
config.py               # 配置管理
.env                    # 本地开发环境变量（已提交用于开发）
```

## 2. 本地开发环境检查

### 启动流程
```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动应用（自动初始化数据库和管理员）
python run.py

# 3. 访问 http://localhost:5002
```

### 验证点
- [ ] SQLite 数据库自动创建在 `instance/exam.db`
- [ ] 管理员账户自动创建（admin/admin）
- [ ] 应用可以正常启动
- [ ] 日志正常输出到控制台

### 环境变量配置
```bash
FLASK_ENV=development         # 开发模式
FLASK_PORT=5002               # 端口
DATABASE_URL=sqlite:///instance/exam.db  # SQLite 相对路径
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin
ADMIN_EMAIL=admin@example.com
```

## 3. Docker 镜像构建检查

### 构建流程
```bash
docker build -t exam-system:latest .
```

### 验证点
- [ ] 镜像成功构建
- [ ] 依赖正确安装
- [ ] gunicorn 正确安装
- [ ] 所有必要目录已创建（uploads/、logs/、instance/）

### Dockerfile 关键检查
```dockerfile
# 使用 Python 3.12
FROM python:3.12-slim

# 工作目录
WORKDIR /app

# 环境变量
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    FLASK_APP=run.py \
    FLASK_ENV=production

# 系统依赖（包含 postgresql-client 用于连接数据库）
RUN apt-get update && apt-get install -y \
    gcc postgresql-client curl libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Python 依赖（包含 gunicorn）
RUN pip install -r requirements.txt && \
    pip install gunicorn

# 暴露端口
EXPOSE 5002

# 启动命令（使用 gunicorn）
CMD ["gunicorn", "--bind", "0.0.0.0:5002", "--workers", "2", \
     "--threads", "2", "--timeout", "120", \
     "--access-logfile", "-", "--error-logfile", "-", \
     "--log-level", "info", "wsgi:app"]
```

## 4. Docker Compose 部署检查

### 部署流程
```bash
# 1. 启动服务（使用默认配置）
docker-compose up -d

# 2. 或使用自定义环境变量
export SECRET_KEY="your-secret-key"
export ADMIN_PASSWORD="your-secure-password"
export POSTGRES_PASSWORD="your-db-password"
docker-compose up -d

# 3. 查看日志
docker-compose logs -f web

# 4. 访问 http://localhost:5002
```

### 验证点
- [ ] PostgreSQL 容器启动成功
- [ ] Web 容器启动成功
- [ ] 数据库迁移自动执行
- [ ] 管理员账户自动创建
- [ ] 健康检查通过
- [ ] 容器重启策略正常

### docker-compose.yml 关键配置
```yaml
services:
  db:
    image: postgres:14-alpine
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-examdb}
      POSTGRES_USER: ${POSTGRES_USER:-examuser}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-exam_password_123456}
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-examuser} -d ${POSTGRES_DB:-examdb}"]

  web:
    environment:
      FLASK_ENV: production
      FLASK_DEBUG: "false"
      SECRET_KEY: ${SECRET_KEY:-change-this-secret-key-in-production}
      DATABASE_URL: postgresql://...@db:5432/examdb
      ADMIN_USERNAME: ${ADMIN_USERNAME:-admin}
      ADMIN_PASSWORD: ${ADMIN_PASSWORD:-admin}
      ADMIN_EMAIL: ${ADMIN_EMAIL:-admin@example.com}
    depends_on:
      db:
        condition: service_healthy  # 等待数据库健康后再启动
```

## 5. 应用初始化检查

### 初始化流程
应用启动时自动执行以下步骤：

1. **数据库连接检查**
   ```python
   db.engine.connect()
   ```

2. **数据库迁移**
   ```python
   from flask_migrate import upgrade
   upgrade()  # 执行所有待执行的迁移
   ```

3. **管理员账户创建**
   ```python
   create_admin_if_not_exists(app)
   ```

### 验证点
- [ ] 数据库连接成功
- [ ] 数据库表结构正确创建
- [ ] 管理员账户成功创建（不存在时）
- [ ] 初始化日志正确输出

### 防止重复初始化
使用全局变量标记，防止 Gunicorn 多 worker 重复初始化：
```python
_app_initialized = False

def initialize_app(app):
    global _app_initialized
    if _app_initialized:
        app.logger.info('应用已初始化，跳过重复检查')
        return
    # ... 初始化逻辑 ...
    _app_initialized = True
```

## 7. 常见问题排查

### 问题 1: 数据库连接失败
**症状**: `OperationalError: unable to open database file`

**解决方案**:
```bash
# 本地开发：确保 instance 目录存在且可写
mkdir -p instance
chmod 755 instance

# Docker: 检查数据库容器是否启动
docker-compose ps db
docker-compose logs db
```

### 问题 2: 管理员账户未创建
**症状**: 无法登录管理员账户

**解决方案**:
```bash
# 检查环境变量
docker-compose exec web env | grep ADMIN

# 查看初始化日志
docker-compose logs web | grep "管理员"

# 手动触发初始化（重启容器）
docker-compose restart web
```

### 问题 3: 数据库迁移失败
**症状**: `alembic.util.exc.CommandError`

**解决方案**:
```bash
# 检查 migrations 目录是否存在
ls migrations/

# 检查迁移文件
ls migrations/versions/

# 查看详细错误
docker-compose logs web | grep -A 10 "迁移"
```

### 问题 4: 端口冲突
**症状**: `Error: bind() failed: address already in use`

**解决方案**:
```bash
# 修改 .env 中的端口
FLASK_PORT=5003

# 或修改 docker-compose.yml
ports:
  - "5003:5002"
```

## 9. 清理和重置

### 完全清理（重新开始）
```bash
# 停止服务
docker-compose down

# 删除数据卷
docker volume rm exam_postgres_data

# 删除本地数据库
rm -f instance/exam.db

# 重新启动
docker-compose up -d
```

### 重置管理员账户
```bash
# 1. 停止容器
docker-compose down

# 2. 删除数据库卷
docker volume rm exam_postgres_data

# 3. 修改环境变量
export ADMIN_USERNAME="newadmin"
export ADMIN_PASSWORD="newpassword"

# 4. 重新启动
docker-compose up -d
```
