# 教师-学生角色系统说明

## 功能概述

系统现在支持两种主要用户角色：**教师**和**学生**。学生可以绑定教师，教师可以查看绑定学生的成绩和考试记录。

## 用户角色

### 学生 (Student)
- 可以参加在线考试
- 可以查看自己的考试记录和成绩
- **可以绑定一位教师**（注册时或后续在个人资料中）
- 绑定教师后，教师可以查看该学生的成绩

### 教师 (Teacher)
- 拥有唯一的教师ID
- 可以查看所有绑定到自己的学生列表
- 可以查看每个学生的成绩统计
- 可以查看每个学生的详细考试记录

## 注册流程

### 学生注册
1. 选择"学生"角色
2. 填写基本信息（用户名、邮箱、密码）
3. **可选**：填写教师ID进行绑定
4. 系统会验证教师ID是否有效
5. 注册成功后自动登录

### 教师注册
1. 选择"教师"角色
2. 填写基本信息（用户名、邮箱、密码）
3. 注册成功后获得唯一的教师ID
4. 可以将教师ID分享给学生进行绑定

## 教师绑定功能

### 在注册时绑定
学生在注册页面可以选择填写教师ID，系统会验证：
- 教师ID是否存在
- 该ID对应的用户是否为教师角色

### 在个人资料中绑定/修改
学生可以随时在个人资料页面：
- 查看当前绑定的教师
- 绑定新的教师
- 更换绑定的教师

### 教师ID查看
教师在个人资料页面可以：
- 查看自己的教师ID
- 一键复制教师ID分享给学��

## 教师查看学生功能

### 我的学生页面 (`/teacher/my-students`)
教师可以查看：
- 学生总数统计
- 活跃学生数（有考试记录的学生）
- 总考试次数
- 平均及格率

### 学生列表
- 学生ID
- 用户名
- 邮箱
- 注册时间
- 考试次数
- 及格率（可视化进度条）
- 查看详情按钮

### 学生详情
点击学生可查看：
- 学生基本信息
- 所有考试记录
- 每次考试的分数和百分比
- 及格/不及格状态
- 考试时间

## API 端点

### 认证相关
- `GET /auth/api/verify-teacher/<int:teacher_id>` - 验证教师ID
- `POST /auth/api/bind-teacher` - 绑定教师

### 教师相关
- `GET /teacher/my-students` - 我的学生页面
- `GET /teacher/api/teacher/students` - 获取学生列表
- `GET /teacher/api/teacher/students/<int:student_id>/submissions` - 获取学生考试记录

## 数据库变更

### users 表新增字段
```sql
ALTER TABLE users ADD COLUMN teacher_id INTEGER;
```

### 关系
- `teacher_id` 外键关联到 `users.id`
- 一个学生可以绑定一个教师
- 一个教师可以有多个学生

## 权限控制

### 学生权限
- 只能绑定一位教师
- 可以查看自己的成绩
- 可以修改绑定的教师

### 教师权限
- 可以查看所有绑定自己的学生
- 可以查看学生的详细考试记录
- 不能修改学生的成绩

### 访问控制
- 只有教师可以访问 `/teacher/*` 路由
- 只有学生可以调用绑定教师API
- 教师只能查看绑定自己的学生数据

## 使用示例

### 场景1：新学生绑定教师
1. 学生注册时填写教师ID（如：5）
2. 系统验证ID 5是否为有效教师
3. 注册成功后自动绑定
4. 教师登录后可在"我的学生"中看到该学生

### 场景2：学生更换教师
1. 学生登录后进入个人资料
2. 在"绑定教师"部分输入新的教师ID
3. 验证成功后点击"绑定"
4. 新教师可以看到该学生，旧教师不再看到

### 场景3：教师查看学生成绩
1. 教师登录后点击导航栏"我的学生"
2. 查看所有绑定学生的统计信息
3. 点击某学生的"详情"按钮
4. 查看该学生的所有考试记录和成绩

## 注意事项

1. **管理员角色**：目前注册时不再提供管理员选项，管理员需要通过数据库或后台手动设置
2. **教师ID唯一性**：教师的ID就是其用户ID，是唯一的
3. **数据隐私**：教师只能看到绑定自己的学生数据，无法查看其他学生
4. **解绑功能**：目前通过绑定新教师实现更换，后续可以添加完全解绑功能
